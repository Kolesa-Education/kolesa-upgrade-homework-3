Утилита ping:

Ping - это утилита для проверки целостности соединения в сетях. При вызове данной утилиты, посылается запрос каждую секунду адресуемому узлу.
Вывод данной утилиты это Echo Reply который предоставит информацию об адресуемой сети.

"64 bytes from 142.250.186.206" означает размер пакета который отправляется по адресу 142.250.186.206
icmp_seq=0 - поле означает icmp_seq используется для отображения порядкового номера отправленных пакетов. То есть если icmp_seq = 3, то будет отправлено 3 пакета
ttl=113 - Поле Time To Live (ttl) используется как счетчик переходов. Каждый переход уменьшает счетчик на 1, и если счетчик равен 0, то пакет умирает. Это используется для контроли продолжительности жизни пакета, ведь мы не хотим чтобы пакеты путешествовали вечно.
time=123 ms - Время приема-передачи, которое потребовалось от отправки пакета эхо-запроса до получения эхо-ответа.

Утилита curl:

CURL - это специальная программа или утилита, которая позволяет делать различные сетевые запросы по различным протоколам данных.
Смысл данной утилиты это отправить определенный запрос, и получить ответ. Так же curl выполняет функцию проверки подключения к URL адресам.

curl -I https://kolesa.kz/ - используется для получения заголовок со страницы kolesa.kz. Так, при выполнения данного запроса, в ответ придут данные с HTTP-header, которое в себе включает информацию как имя сервера, дату документа, HTTP версию и многое другое.
Так например при выполнения команды, curl -I https://kolesa.kz/ , одними из выходных данных будут HTTP/2 200, которые обозначает status code сервера, а именно что все OK, и соединение установлено, а так же server: nginx, который показывает какой сервер запущен на kolesa.kz.
Флаг -I обозначает -Include
