2. Изучение утилиты ping.
Утилита ping нужна для проверки доступности и диагностики качества сети.
Команда выводит:
- ip адрес, который проверяется
- число отправленных байт
- icmp_seq отображает номер последовательности эхо-запроса и эхо-отклика
- ttl (time to life, время жизни в сети) - количество устройств в сети, через которое проходит пакет
- time - время, через которое был получен ответ от проверяемого сервера
По завершению работы команда выводит статистику по отправленным/полученным пакетам, время приема/передачи

3. Изучение утилиты curl.
curl (client url) необходима для работы с URL страницами и передачи файлов.
Команда выводит:
- версию сетевого протокола HTTP и код ответа (статус код). В частности, "HTTP/2 200" означает, что версия сетевого протокола 2, а код ответа - 200 (успех)
- веб-сервер, на котором работает сервис. В частности, "server: nginx" означает, что используется веб-сервер nginx
- дата и время получения ответа
- тип возвращаемых данных и кодировку
- размер возвращаемых данных
- тип соединения, повтороное использование соединений HTTP
- установленные куки и время их "просрочки"
Флаг "-I" необходим для получения только заголовков HTTP, без содержимого страницы
